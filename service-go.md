# Roadmap: Подключение Go Backend к OpenAI API

## Цель

Создать AI-сервис на Go, который:

- принимает запрос от клиента;
- отправляет его в OpenAI;
- получает ответ;
- возвращает его клиенту.

После этого постепенно добавить XML Prompting, Structured Output и RAG.

---

# Этап 1. Получить API Key

## Цель

Получить возможность обращаться к OpenAI API.

## Что сделать

1. Зарегистрироваться в OpenAI.
2. Создать API Key.
3. Сохранить его в `.env`.

```env
OPENAI_API_KEY=sk-xxxxxxxxxxxxxxxx
```

## Результат

✅ Есть API Key.

---

# Этап 2. Подключить OpenAI к Go

## Цель

Научиться отправлять самый простой запрос.

Архитектура

```text
Client
    │
    ▼
Go Backend
    │
    ▼
OpenAI API
```

## Что сделать

- установить SDK OpenAI (или использовать `net/http`);
- создать клиента;
- отправить простой запрос.

Например:

```
Привет! Кто ты?
```

## Результат

✅ Go умеет отправлять запросы в OpenAI и получать ответ.

---

# Этап 3. Создать endpoint

Например

```
POST /ai/chat
```

Body

```json
{
    "message":"Привет!"
}
```

Архитектура

```text
Client
      │
      ▼
POST /ai/chat
      │
      ▼
Go Backend
      │
      ▼
OpenAI
      │
      ▼
Ответ
```

## Результат

✅ Backend стал посредником между клиентом и OpenAI.

---

# Этап 4. Добавить System Prompt

## Цель

Настроить поведение модели.

Вместо

```
User:
После оплаты приложение зависло.
```

использовать

```
System:
Ты опытный QA-инженер.
Отвечай только на русском языке.
Не придумывай факты.

User:
После оплаты приложение зависло.
```

## Результат

✅ Модель начинает отвечать в соответствии с инструкциями.

---

# Этап 5. Добавить XML Prompting

Теперь пользовательский запрос оформляется в XML.

Пример

```xml
<prompt>

    <task>
        Сгенерируй bug report.
    </task>

    <input>
        После нажатия кнопки "Оплатить" приложение зависло.
    </input>

</prompt>
```

## Результат

✅ Начинаете использовать XML Prompting.

---

   ▼

✅ Первый AI-сервис готов.

# Этап 7. Добавить Structured Output

Вместо обычного текста получать JSON.

Например

```json
{
    "title": "...",
    "steps": [],
    "actual_result": "...",
    "expected_result": "...",
    "severity": "Critical"
}
```

## Результат

✅ Frontend получает готовую структуру данных.

---

# Этап 8. Добавить Streaming

Получать ответ постепенно.

```text
OpenAI
 │
 ├── "Bug..."
 ├── "Report..."
 ├── "..."
```

## Результат

✅ Пользователь видит генерацию ответа в реальном времени.

---

# Этап 9. Подключить RAG

Архитектура

```text
Request
    │
    ▼
Backend
    │
    ├── Поиск в pgvector
    │
    ├── Получить релевантные документы
    │
    ├── Добавить документы в Context
    │
    ▼
OpenAI
```

Пример XML

```xml
<prompt>

    <context>
        Информация из базы знаний...
    </context>

    <task>
        Сгенерируй bug report.
    </task>

    <input>
        Пользователь нажал кнопку "Оплатить".
        Приложение зависло.
    </input>

</prompt>
```

## Результат

✅ Ответы модели становятся точнее благодаря базе знаний.

После прохождения всех этапов вы сможете разработать AI-сервис, который:

- принимает Test Case;
- ищет релевантную информацию в базе знаний (RAG);

- добавляет System Prompt;
- отправляет запрос в OpenAI;
